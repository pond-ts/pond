# Pan, zoom & range selection

Three related mechanisms move the **view range** ‚Äî the slice of the x axis a
chart shows. They compose but are independent: drag-to-select a range, wire
that into a controlled zoom, and/or enable free pan/zoom. All of them speak
the same neutral range contract: a `[start, end]` pair in axis units.

Chapter 6 teaches the select-to-zoom loop as a payoff; this page is the full
prop reference. See
[Learn charts, chapter 6](../../learn-charts/cursors-readouts-zoom) for
the narrative and the [`Cursors/Region`](pathname:///storybook/?path=/story/cursors-region--drag-to-select)
Storybook group for the per-variant walk.

## Range selection ‚Äî `cursor="region"`

Setting `cursor="region"` and providing `onRegionSelect` makes the region
cursor **draggable**: the user drags across the plot and, on release, you get
the selected range once.

```ts
cursor="region"
cursorSequence?: Sequence | BoundedSequence; // bucket snapping (time axis only)
onRegionSelect?: (range: readonly [number, number]) => void; // [lo, hi] on release
```

- **`onRegionSelect` fires once, on release,** with `[lo, hi]` ‚Äî always
  ordered low-to-high regardless of drag direction. Providing it is what turns
  the region cursor draggable; it's a **notification** ‚Äî the container never
  zooms itself in response.
- **`cursorSequence`** snaps the shaded band to buckets (whole minutes, whole
  sessions) as the drag extends, and is **time-axis only** ‚Äî a "minute" is
  meaningless on a value axis. With no `cursorSequence` the region cursor is
  the degenerate free-drag case (pixel-precise, no snapping), which is what a
  value-axis selection uses.
- Range selection works on a **time or value** x axis; it's excluded on a
  category axis.

## The select-to-zoom loop

Zooming is `onRegionSelect` ‚Üí your state ‚Üí controlled `range`. The container
draws whatever `range` you give it; the selection callback proposes a new one;
you decide whether to apply it. A "reset" is just setting `range` back to the
full extent.

```tsx
const [range, setRange] = useState(full);

<ChartContainer
  range={range}
  cursor="region"
  onRegionSelect={(r) => setRange(r)}
>
  {/* ‚Ä¶ */}
</ChartContainer>;
```

Because the container never zooms on its own, the same `onRegionSelect` can
drive anything a range means in your app ‚Äî a filter, a fetch, a linked chart ‚Äî
not only a zoom.

## Free pan & zoom ‚Äî `panZoom`

Separately from range-select, `panZoom` enables **direct manipulation**: drag
the plot to pan, wheel to zoom around the cursor. It's a **three-way mode** ‚Äî
choose whether the plot captures pan, both, or neither:

| `panZoom`                    | Gestures                                                 |
| ---------------------------- | -------------------------------------------------------- |
| `'none'` / `false` (default) | Neither ‚Äî the plot ignores drag and scroll.              |
| `'pan'`                      | Drag pans; the wheel is left alone (page still scrolls). |
| `'panZoom'` / `true`         | Drag pans **and** wheel zooms around the cursor.         |

| Prop                | Type                                      | Default  | Purpose                                                                                           |
| ------------------- | ----------------------------------------- | -------- | ------------------------------------------------------------------------------------------------- |
| `panZoom`           | `'none' \| 'pan' \| 'panZoom' \| boolean` | `'none'` | Which gestures the plot captures (table above). Boolean is the back-compat shorthand.             |
| `bounds`            | `[number, number]`                        | ‚Äî        | **Outer extent** ‚Äî pan/zoom can't move the view outside it. Zoom-out ceiling; omit for no limit.  |
| `minDuration`       | `number`                                  | `1`      | Zoom-**in** floor ‚Äî the minimum visible duration in ms.                                           |
| `onTimeRangeChange` | `([start, end]) => void`                  | ‚Äî        | Fires on pan/zoom with the new range. Wire back to `range` for controlled; omit for uncontrolled. |
| `range`             | `[number, number] \| TimeRange`           | ‚Äî        | The controlled view range (shared by all three mechanisms).                                       |

**Bounding the reachable window.** `bounds` and `minDuration` fence the view
between an **outer** and an **inner** limit: `bounds` is the `[min, max]` the
view can never move outside ‚Äî pan into an edge stops there (keeping its span),
and zoom-out is capped at the whole span ‚Äî while `minDuration` is the narrowest
visible window (the zoom-in floor). Set both and a gesture can neither scroll
off into empty time nor zoom past a useful grain. Seed `range` within `bounds`.
(On a trading-time axis, `bounds` clamps in wall-clock ms ‚Äî a sensible outer
limit; the per-session pan/zoom math already holds the trading span at each
calendar edge.)

**Controlled vs. uncontrolled matters here.** Uncontrolled (`panZoom` on, no
`onTimeRangeChange`), the container holds the view internally and seeds it from
`range` whenever it isn't actively holding one ‚Äî but later `range` changes are
then ignored so they can't fight the user's pan. To drive the range externally
‚Äî or to **follow a live sliding window** ‚Äî use controlled mode: handle
`onTimeRangeChange` and feed `range` yourself.

## Sharp edges

- **The container never zooms itself.** `onRegionSelect` and
  `onTimeRangeChange` are notifications; a controlled `range` is the only thing
  that actually changes what's shown. This is deliberate ‚Äî a range means
  different things in different apps.
- **`cursorSequence` is time-axis only.** Bucket snapping needs a temporal
  grid; value-axis and category selections don't take one.
- **Uncontrolled `panZoom` stops tracking `range`** once the user grabs it ‚Äî
  if you need the view to follow live data, go controlled.
- **`bounds` constrains _gestures_, not the initial `range`.** Only pan/zoom
  output (and any range routed through the container) is clamped ‚Äî a `range`
  seeded outside `bounds` shows as-is until the first gesture pulls it in. Seed
  it within `bounds` yourself.
- **`bounds` wins over `minDuration` if they contradict.** A `bounds` span
  narrower than `minDuration` clamps to the (narrower) extent; keep
  `minDuration ‚â§ bounds` width.
- **Mouse-only** ‚Äî pan is drag, zoom is wheel; no keyboard or touch gestures
  yet.
